## サービス概要
MabaTalkは、発声や身体操作が困難で、まばたきによるYes/Noの反応のみで意思表示を行う方のための、「メッセージ確認 × 回答履歴共有」ができるコミュニケーション支援Webアプリです。

要介護者が最小限の負担で意思を示し、家族・介護スタッフが迷わず本人の気持ちや体調を理解・共有できることを目的としています。

## このサービスへの思い・作りたい理由
- 妻の義父が脳卒中で倒れ、介護施設で寝たきりの状態となりました。
- 言葉を発することはほとんどできず、意思を伝える手段は「目を閉じる・開く」といったまばたきだけになります。
- 体調や要望を尋ねても、「暑い」「寒い」「痛い」といった変化をすぐに把握できず、表情だけでは判断が難しいため、もどかしさを感じる状況が続いていました。
- この状況を少しでも改善するため、本人が最小限の負担で意思を示せて、家族・介護スタッフが迷わず理解できるコミュニケーション手段を作りたいと考え、本サービスの開発を決めました。
- 本サービスを通して、家族・介護スタッフの負担を減らすだけでなく、本人が「伝えたい気持ち」や「生活の選択肢」を少しでも取り戻すきっかけになればと考えています。

## サービスの利用イメージ
1. 家族・介護スタッフがアプリを起動し、メッセージカテゴリを本人に見せます
2. 項目を順に指し示し、伝えたい内容があれば本人にまばたきしてもらいます
3. 家族・介護スタッフが項目を選択し、確認画面で「はい/いいえ」を本人に確認します
4. 「はい」が選択された内容が履歴として保存されます

未ログイン状態でも、メッセージの選択と意思表示の確認まで行うことができます。
ログイン時のみ、選択結果が履歴として保存されます。

## 本サービスの特徴
- 利用者ごとにメッセージ項目を柔軟にカスタマイズできる構成
- 文脈（カテゴリ）に応じて、確認内容を段階的に提示
- 選択された内容を履歴として残し、後から振り返りや共有が可能

固定された選択肢では対応しづらい状況でも、利用者の状態や環境に合わせた意思疎通を行えることを重視しています。

## 機能候補
▪️実装済み機能（MVP）
- カテゴリ別のメッセージ選択機能
- 選択結果のログ保存機能
- カスタムカテゴリ・項目の作成機能
- ユーザー認証機能

▪️本リリースまでに実装したい機能
- 自動音声読み上げ機能(Web Speech APIを使用予定)
- メッセージ項目の編集・削除・並び替え機能
- WebGazer.jsを使った視線選択
- 写真・動画アップロード機能
- カレンダー・メモ機能
- 家族アカウント管理
- 複数人の家族によるログ共有機能
- 初期確認用の質問テンプレート機能（検討中）
- AIによる回答ログの分析・可視化(実装方法は検討中)
